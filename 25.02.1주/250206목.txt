책무: 금융회사 또는 금융회사 임직원이 준수해야하는 사항에 대한 내부통제 및 위험관리의 집행 및 운영에 따른 책임

# 금융용어
채널계: 사용자가 접속하는 다양한 채널의 데이터를 관리하는 시스템 / 모바일 뱅킹, 인터넷 뱅킹, ATM, 콜센터, 현업 단말기 등
계정계: 고객 거래 데이터, 계좌, 계정을 처리하는 핵심 시스템 / 공통, 수신, 여신, 신탁, 보험, 카드, 외환, 대행업무 등으로 구성
정보계: 계정계에서 받은 데이터를 바탕으로 거래 활동 및 성과를 측정하고 분석하는 시스템 / 데이터 접근 속도 중요 / 수익 관리, 고객관계관리, 성과관리, 위험관리 등
운영계: 서비스 운영을 위한 유지보수, 통합관제, 모니터링등의 시스템 / IT인프라(HW, SW), 네트워크, 보안 영역 등
대외계: 금융기관이 외부기관과의 연계되는 시스템 / 은행 공동망, 제휴 기관, 금융 결제원 등 / 안정성과 정확성 중요
오픈뱅킹: 다른 금융기관(핀테크) 서비스임에도 내 금융정보를 조회하거나 이체하는 등의 서비스를 제공해주는 것 / 하나의 앱으로 여러 금융기관 서비스 이용
마이데이터: 각종 공공기관이나 금융권에 분산된 고객정보를 하나의 플랫폼에서 통합 관리해주는 서비스 / 소비 습관, 자산 현황 등을 종합 분석해 금융상품 추천
PT(Presentation Tier): 채널계, 뷰 역할, 사용자가 입력한 정보를 BT에 전달
BT(Business Tier): 계정계, 비지니스 로직 영역. 데이터 처리, 비지니스 규칙 포함
DT(Data Tier): 정보계, 데이터 저장, 관리, 처리

## 코스콤 금융IT 용어사전
가용성(Availability): 서버와 네트워크, 프로그램 등의 정보 시스템이 정상적으로 사용 가능한 정도 = 가동률
개인정보 비식별화: 정보 집합물에서 식별정보를 제거해 개인을 특정할 수 없게 가명화, 익명화하는 방법과 조치
데이터 3법: 개인정보 보호법, 정보통신망법, 신용정보 보호법
	개인정보(특정 가능) - 구체적인 동의 후 활용 가능 / 가명 정보 - 동의 없이 통계, 연구, 공익적 기록 등 가능 / 익명정보 - 자유
공공 마이데이터: 공공기관이 보유한 자신의 행정정보를 자신이 원하는 곳에 제공하도록 요구할 수 있는 서비스
노드 인증: 관리 서버와 데이터 콜렉터가 보안 방식으로 서로 통신하는지 확인하기 위해 사용되는 기술
라우팅(routing): 네트워크에서 통신 데이터를 보낼 경로를 선택하는 과정
망 분리: 외부 공격으로부터 내부 자료 보호를 위해 업무용 내부 망과 인터넷 망을 분리하는 것
락인(Rock-in): 다른 제품으로 소비 전환하지 않고 기존 서비스를 꾸준히 유지하는 현상 / 사용자 고착, 자물쇠 효과
미들웨어: 양 쪽을 연결하여 데이터를 주고 받을 수 있게 매개 역할을 하는 소프트웨어
부인방지: 메시지 송수신이나 통신 처리 후 그 사실을 사후에 증명함으로써 사실부인을 방지하는 보안 기술
블록체인: 네트워크 내 모든 참여자가 공동으로 거래 정보를 검증, 기록, 보관함으로써 공인된 제3자 없이도 거래 기록의 신뢰성을 확보하는 기술 / 작업증명(Proof of Work)
IaaS (서비스형 인프라): 자원만 주는 것 / 네이버 클라우드 등
PaaS (서비스형 플랫폼): 플랫폼을 가상화해서 웹으로 제공하는 형태(가상 인프라를 통해 미들웨어, 런타임, OS가 포함)
SaaS (Software as a Service, 서비스형 소프트웨어): 설치하지 않고 온라인으로 실행 가능한 소프트웨어
		 / 오피스365, 구글 독스, 인사관리 소프트웨어, 전자 상거래 시스템, 고객 관리 툴, 통합개발환경 서비스 등
AES (Advanced Encryption Standard, 대칭키 암호화): 암호화키 = 복호화키
	데이터 전송, 파일 암호화, VPN 등 대량 데이터 암호화 / HTTPS를 통해 전송된느 데이터 암호화, Wi-Fi WPA2 보안 프로토콜 등
RSA (Rivest-Shamir-Adleman, 공개키 암호화): 암호화키=공개키, 복호화키=개인키
	작은 데이터의 안전한 전송, 키 교화, 인증서 서명, SSL/TLS 인증서에서 클라-서버 간 키 교환
	- 공개키로 암호화된 데이터는 비밀키로만 복호화 가능
	- 비밀키로 서명된 데이터는 공개키로만 검증 가능
	- 공개키: 일반적으로 서버가 소유 -> 클라이언트에게 배포
		-암호화: 서버로 데이터를 보낼 때 공개키로 암호화 (서버의 개인키 없이 복호화X)
		-인증: 클라이언트는 서버가 제공하는 공개키를 사용해 서버의 신원 확인, 공개키가 인증서와 함께 제공, 이 인증서를 통해 서버의 신뢰성 검증
	- 개인키: 서버가 소유, 외부에 공개X, 암호화된 데이터를 복호화하고, 서버의 신원을 증명
		-복호화: 클라이언트가 서버의 공개키로 암호화한 데이터를 서버는 자신의 개인키로 복호화할 수 있음
		-디지털 서명: 서버는 개인키로 데이터를 서명하여, 클라이언트에게 자신이 신뢰할 수 있는 서버임을 증명

ESB: 서비스들을 컴포넌트화된 집합으로 묶는 핵심 미들웨어. 비즈니스 프로세스 환경에 맞게 설계 및 전개할 수 있는데 아키텍쳐 패턴
	개발자가 서비스를 직접 개발하고 적절한 api를 통해 서비스간 통신하는데 도움이 되는 인프라 구조 소프트웨어
End Point: API가 서버에서 리소스에 접근할 수 있게 하는 URL
	서비스의 실제 비즈니스를 수행하기 위해 연결하는 연결고리. 바인딩할 경우 클라이언트에서 웹 서비스 호출 가능
EAI: Enterprise Apllication Integration: 기업 내부 이기종 시스템 연계
FEP: Front End Processor: 메인프레임에서 통신 과부하를 줄이기 위해 전처리 작업하는 과정. / 금융권에서는 B2B 연계
MCA: Multi Channel Architecture: 기업 내부 동기종 또는 유사기종 시스템 연계(은행에서의 여신-수신 업무 연계 등)
SOA: Service Oriented Architecture: 기존 앱의 기능들을 비즈니스적 의미를 가지는 기능 단위로 묶어 컴포넌트 단위로 재조합 후, 업무기능 구현 앱을 만드는 소프트웨어 아키택처
ERP: Enterprise Resource Planning, 전사적 자원 관리: 기업 내부의 모든 인적, 물적 자원을 효율적으로 관리할 수 있는 기업의 통합 정보 시스템
CRM: Customer Relationship Management, 고객 관계 관리: 고객 정보를 수집, 분석해 고객의 성향에 맞는 관계를 구축하는 것
SCM: Supply Chain Management, 공급망 관리: 유통 생산에 관련된 과정을 관리하는 것
BPR: Business Process Reengineering, 업무 재설계: 기업의 활동이나 업무의 전반적 흐름을 분석하고, 경영 목표에 맞게 조직과 사업을 최적으로 다시 설계해 구성하는 것
ISP: Information Strategy Planning, 정보전략계획: 조직의 경영계획과 목표를 지원하기 위한 정보시스템의 비전 계획
SM: System Management, 시스템 관리: 대기업 혹은 공공기관에서 제공해주는 장소에서 이미 개발된 시스템의 유지보수를 맡거나 추가 개발하는 것
ECM: Enterprise Content Management: 전사적 콘텐츠 관리: 조직 내 처리 업무에 관한 콘텐츠나 문서를 보관, 전달, 관리에 이용하는 기술. 비구조화 정보가 어디있든, 관리할 수 있는 형태의 환경으로 만듦
	WCMS: Web Content Management System, 웹 콘텐츠 관리 시스템: 비전문가도 웹 콘텐츠를 효과적으로 관리해주는 솔루션
	DAM: Digital Asset Management, 디지털 자산관리: 디지털 자산을 생성, 저장, 검색하고 사용권한을 관리하기 위한 비즈니스 시스템
CDC솔루션( Change Data Capture): 마지막 추출 후 변경된 데이터만 골라내는 기술. 시스템 로드를 줄이고 작업 생산성 향상 / 주로 DB 로그 기반 CDC
AS-IS분석: (있는 그대로) 현재 업무 프로세스 분석 / TO-BE분석: (미래의) 개선될 업무 프로세스에 대한 분석
batch: 사용자의 개입없이 실행을 스케쥴링할 수 있는 작업. 프로그램 흐름에 따라 순차적으로 자료를 처리하는 방식
batch process : 일괄 처리. 주기적으로 대량의 반복적인 데이터 작업을 완료하기 위해 사용하는 방식 / 보험회사 - 우편번호 가져오기 예시
	- 비 대화형 처리: 사용자와 상호작용X
	- 대량 데이터 처리
	- 스케쥴링: 낮은 트래픽 시간대에 실행하여 시스템 부담 최소화
	- 오류 처리 및 로그: 수행 작업에 대한 로그

EEC (Elliptic Curve Cryptography, 타원 곡선 암호화)
Hashing: 암호화만 가능(복호화X)
PKI (Public Key Infrastructure, 공개키 기반 구조): 인증서를 통해 서로 신뢰할 수 있게 하는 구조
	인증서, 인증기관(발급 및 관리), 등록기관(신원확인, 고객 테이터 유지 등 인증기관의 입증 대행), 디렉토리, 사용자
SSL/TLS 
HSM (Hardware Security Module, 하드웨어 보안 모듈)
TLS, SSL : 통신 프로토콜
MFA: 다중 인증
세션 하이재킹
SHA: 미국 NSA가 제작하고 미국 국립표준기술연구소(NIST)에서 표준으로 제작한 해시 암호 알고리즘 / 인터넷뱅킹 등
CBC, GCM (암호 운영 모드)
MCA(MCI): 다양한 채널마다 상이한 통신방식과 전문들을 동일한 규격으로 통일시켜주는 것
On Demand Batch: 배치작업을 스케쥴링이 아닌, 사용자 요청시에만 하는 것

<프로젝트 용어>

결함 : 통합테스트를 했을때 프로젝트가 요구사항이나 사양을 충족하지 못하여 수정 또는 교체가 필요한 불완전한 상태
프리징 : 프로젝트/시스템에 영향을 주는 변경사항에 대해 더이상 수용하지 않는 시점을 의미함. 프리징 기간에는 로컬에서만 변동하고 서버로 업데이트 하지않는다.
이슈 : 분쟁의 대상이 되는 쟁점 또는 해결되지 않고 논의중에 있거나 상반되는 견해를 가진 문제.
QA(Quality Assurance) :  품질보증. 프로젝트가 관련된 품질기준을 충족시킨다는 것을 확신시키기 위하여, 정기적으로 프로젝트의 전반적인 수행을 평가하는 절차.
WBS(Work Breakdown Structure) : 작업분류체계.  프로젝트의 전체의 요소들을 인도물 위주의 그룹으로 묶는 것. 하위수준으로 내려갈수록 프로젝트 업무가 점진적으로 상세하게 정의된다.
PMO(Project Management Office)  : 프로젝트 관리업무 전담 조직
QA(Quality Assurance) : 품질보증담당자
R&R(Role and Responsibilities) :  프로젝트 내에서의 프로세스 및 조직의 구성원들이 수행해야할 역할과 그 역할의 수행에 따른 책임관계의 정립
WBS(Work Breakdown Structure) : 작업분할구조도. 프로젝트 목표를 달성하기 위해 필요한 활동과 업무를 세분화하는 작업.

<금융, 회계 용어>
ARK(Agile, Reliable, Keen) : 직역이 아님. 보험개발원에서 공동개발 중인 부채평가시스템.
BEL(Building Block Approach) : 최적추정(가정) 부채.
CF(Cash Flow) : 현금흐름
COA(Chart Of Account) : 계정과목일람표
CoC(Cost Of Capital) : 자본비용. 자본을 사용하기 위해 지불해야 하는 대가로, IFRS17에서 위험조정을 계산하는 방법중 하나. 자본비용율에 요구자본비용을 곱하여 위험조정값을 계산함.
CSM(Contractual Service Margin) : 보험계약마진. 보험계약집합에 대한 자산 또는 부채장금액의 구성요소러소, 집합 내 보험계약에 따라 서비스를 제공하면서 인식하게 될 미실현이익을 나타냄.
CSM이 +면 이익, -면 손실
DATA MART : 데이터 저장소. 특정 사용자가 관심을 갖는 데이터들을 담은 비교적 작은 규모의 데이터 웨어하우스의 형태.
G/L(Heneral Ledger) : 총계정원장. 기업 회계상 모든 계정의 수입과 지출 등을 수록한 장부
GOC(Group Of Contracts) : 계약그룹(집합)
IFRS(International Financial Reporting Standards) : 국제회계기준. 기업의 회계 처리와 재무제표에 대한 국제적 통일성을 높이기 위해 국제회계기준위원회에서 마련해 공표하는 회계기준.
K_ICS(Korea-Insurance Capital Standard) : 신지급여력제도. 새로운 보험사 지급여력제도로 2021년 도입되는 IFRS17에 따라 기존 원가평가방식에서 시가평가방식으로 재무건전성을 평가하는 제도.
K-IFRS(KOREA - International Financial Reporting Standards) : 한국채택국제회계기준. 계정과목을 세세하게 구분하지 않고 포괄적으로 기재함.
LAT(Liability Adequacy Test) : 보험부채적정성평가. 보험계약으로부터 발생할 미래 현금유입과 유출액을 현재 가치로 바꿔 책임준비금이 적정한 지를 따지는 제도.
RA(Rist Adjustment) : 위험관리
SAP(Statutory Accounting Principles) : 감독회계기준
ITO : 전산용역비
CSR :  Client Server Request 인듯
채널 : 상품을 파는 경로.
AS IS, TO BE : 현재는 이렇고, 미래의 이상
TM : Tele Marketing. 전화로 영업하는거. I/B는 인바운드. 콜센터로 전화오는거.
                                                                       O/B는 아웃바운드. 콜상담원이 전화거는거.
GA : 대리점. 회사에서 직접 운영하는것은 지점 이라 하고, 흥국에서 몇개, 삼성에서 몇개, 한화에서 몇개 이런식으로 다양한 회사의 상품을 운영하는데를 대리점이라고 한다.
AD( Activity Driver) :  활동원가동인
RD(Resource Driver) : 전표원가동인?
약대 : 약관대출
수보 : 수입보험료
유지율 : 해지율의 반대
전사 : 회사 전체
공통비 : 한영역에서만 쓰지않고 공통적으로 쓴 비용.
판관비 : 판매비와 관리비. 매출액 = 매출원가 + 판관비
증번 : 증권번호.
배부 : Based Costing
이수관 : 관리점 변경
이연 : 특정한 사정으로 인해 손익의 귀속이 다음 회기로 연장되는 것. 
예상 : 이연의 반댓말. 내년에 귀속되어야 할 손익이 올해의 손익으로 처리되는 것.
감가상각 : 시간의 흐름에 따른 자산의 가치 감소(감가)를 회계에 반영하는 것(상각)
선수금 : 회사에서 제품이나 용역을 제공하기 전에 미리 받은 금액. 부채이지만 현금 유동성이 좋아진다.

ERD(Entity Relationship Dagram) : 개체-관계 다이억그램. 어플리케이션 개발자 간의 자료를 공통적으로 이해할 수 있도록 하는 매체가 됨.
ETL(Extraction, Transformation, Loading) : 데이터의 추출, 변환, 적재를 일컫는 말.
SI(System Integration) : 시스템을 구성하는 기기, 소프트웨어 및 관련하는 기술을 선정하고 또한 정비하여 정보처리시스템으로서 종합적으로 구축하는 행위
WAS(Web Application Server) :  HTTP를 통해 컴퓨터나 장치에 애플리케이션을 수행해주는 미들웨어
