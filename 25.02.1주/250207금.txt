책무: 금융회사 또는 금융회사 임직원이 준수해야하는 사항에 대한 내부통제 및 위험관리의 집행 및 운영에 따른 책임

프린트박스 11:20 수령 / 제안요약서(컬러) 13 / 제안서 흑백 2

여신: 고객에게 신용을 주는 일 / 고객을 믿고 돈을 빌려주는 행위(대출)
수신: 고객에게 신용을 받는 일 / 고객이 은행을 믿고 돈을 맡기는 행위(예금/적금)
PB (Private Banking): 규모가 크고 복잡한 재산을 가진 개인을 상대하는 1:1 자산 운용 상담

업무 예시
- 책임자를 지정하여 총괄적으로 수행하는 업무: 내부통제등 총괄관리, 내부감사, 준법감시, 위험관리, 자금세탁방지, 정보보안, 내부회계관리 등
- 영업 관련 부문별 업무: 여신,수신, 투자매매, 중개, 집합투자,신탁, 보험계약체결,인수, 신용카드업, 연금(개인연금, 퇴직연금) 등
- 경영관리 관련 업무: 인사,교육, 보수, 고유자산운용, 건전성 관리, 업무위수탁, 자회사 관리, 광고 등

DR (Disaster Recovery, 재해 복구): 중단 발생 후 가용한 툴과 기술을 이용해 IT 서비스를 복구하는 일련의 단계
BC (Business Continuity, 비즈니스 연속성,회복성): 광범위한 형태의 데이터 보호. 복원을 포함해 비즈니스 운영을 지속하기 위한 절차 포함
CDP(Continuous Data Protection, 지속적 데이터 보호): 모든 데이터 변경본을 자동 복사해 관리자가 언제든 데이터를 복원할 수 있게 끔하는 데이터 백업
HA(High Availability, 고가용성): 재해 중 비즈니스를 지속하는데 도움이 될 수 이쓴 기술 특성. 손상 시 failover하여 원래 시스템 대체. 보호 기능은 아님
Repllication(복제): 재해 시 최신 데이터를 복원할 수 있게, 다른 위치로 데이터를 복사하는 프로세스

RFP (Request For Proposal): 클라이언트가 업체에 요구한 사항 / 프로젝트 목적, 범위, 예산, 일정, 기술적 요구사항 등
Scope (범위): 프로젝트에서 수행될 구체적인 작업 / 무엇부터 어느 지점까지 포함하는가의 범위
MileStone: 프로젝트 일정에서 중요한 지점이나 목표
KickOff: 프로젝트의 공식적인 시작
WBS (Work Breakdown Structure, 업무 분류 체계): 프로젝트를 더 작고 관리하기 쉬운 작업들로 나누는 방법 / 팀, 부서 역할 명확화
API (Application Programming Interface): 소프트웨어 시스템간 연결하는 방식
UAT (User Acceptance Testing): 고객이 직접 시스템을 테스트하여 요구사항을 충족하는지 확인하는 단계
회귀테스트 (Regression Test): 새로운 기능 추가나 버그 수정후, 기존 기능들이 여전히 제대로 작동하는지 확인하는 테스트
Go-Live: 공식적으로 런칭되어 실제 사용되기 시작하는 시점
SLA (Service Level Agreement): 업체가 시스템을 어떻게 지원할 것인지 정의한 계약
Patch: 문제를 해결하거나 소소한 기능을 추가하기 위한 비교적 작은 업데이트 / 보안 강화, 버그 수정, 성능 개선 등을 위해 주기적으로 적용
Scalability (확장성): SI 시스템이 사용자나 데이터를 처리할 수 있는 능력
무결성: 데이터 값이 정확한 상태
정합성: 데이터의 정확 유무와 별개로 데이터들의 값이 서로 일치하는 상태
클라우드 컴퓨팅: 인터넷을 통해 컴퓨팅 자원을 필요에 따라 사용할 수 있게 해주는 서비스 / 서버, 저장소, DB 등 포함
NAC (Network Access Control): 엔드포인트 보안기술 / 사용자 또는 시스템 인증 및 네트워크 보안을 통합하는 보안 접근 방식
NAT (Network Address Translation): 사설 IP를 공인 IP로 변환하는 서비스
NAS (Network Attached Storage): 네트워크 환경 데이터 스토리지
