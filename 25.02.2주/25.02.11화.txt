책무: 금융관계법령 등에 따라 금융회사 또는 금융회사 임직원이 준수해야하는 사항에 대한 내부통제 및 위험관리의 집행 및 운영에 대한 책임

PRA (Robotic Process Automation, 로보틱 처리 자동화): 사람이 반복적으로 처리하는 업무를 로봇 소프트웨어를 통해 자동화 하는 솔루션
	정보통신 시스템간 손쉬운 연결을 통해 직원들이 반복적인 작업에서 벗어나, 창의적인 업무 수행이 가능, 인력 감축
U2L: 주 전산시스템을 유닉스에서 리눅스로 전환하는 것
오픈 아키텍처: 플랫폼 상 기업과 고객, 고객과 고객을 연결해주기 위해 필요한 기반인 시스템.
	기업의 시스템 또는 콘텐츠를 개방하고 공유함으로써 외부의 개발자 및 사용자들이 새로운 창작물을 만들거나 문제를 해결할 수 있음
오픈소스 SW (OSS): SW개발자의 권리를 보장하면서, 소스코드를 누구나 열람할 수 있게 하는 공개된 개발 모델.
	 소스코드 사용,복제,배포,수정 허용. 사용자에게도 라이선스를 줌으로써 권리와 의무를 부여
블록체인: 네트워크 내 모든 참여자가 공동으로 거래 정보를 검증,기록,보관함으로써 '공인된 제3자'없이 거래 기록의 신뢰성을 확보하는 기술.
	특정시간(10분) 발생된 모든 거래정보가 기록된 Block을 생성, 모든 구성원에게 전송해서 같은 거래장부 사본을 나눠 보관하는 분산원장 기술
	공개키 암호화 기술과 작업증명(Proof-of-Work) 매커니즘을 사용해 거래 기록의 신뢰성을 확보
부인방지: 메시지의 송수신이나 교환 후, 또는 통신이나 처리 실행 후 그 사실을 사후에 증명함으로써 사실 부인을 방지하는 보안 기술.
미들웨어: 양 쪽을 연결해 데이터를 주고 받을 수 있도록 매개 역할을 하는 소프트웨어 / TP 모니터 (Transaction Processing Monitor)
라우팅: 네트워크 안에서 통신 데이터를 보낼 경로를 선택하는 과정 / 패킷 스위칭 네트워크에서 패킷 전달 및 라우터,브릿지,게이트웨이,방화벽,
	스위치라 불리는 중간 노드를 거쳐 논리적으로 주소가 부여된 패킷의 변환을 총괄
도메인 네임: 네트워크 상에서 컴퓨터를 식별하는 호스트명
데이터 마트: DW (Data Warehouse, 중앙 집중식 데이터 집합체; 활용 전 데이터)에서 데이터를 꺼내 사용자에게 제공하는 역할. 
	사용 목적에 맞는 접근성과 사용성을 강조. 대개 특정한 조직 또는 팀에서 사용하며 전략적이고 명확한 목표를 가짐
데이터모델링: 현실 세계의 데이터를 컴퓨터 세계의 데이터베이스에 옮기는 변환 과정 / 추상화, 모형화, 단순화, 명확화
개념적 데이터모델링: 조직, 사용자의 요구사항을 찾고 분석하며, 특정 자료의 중요도와 유지 여부 결정 / ERD
논리적 데이터모델링: 업무의 구체적인 흐름. 비즈니스 정보의 논리적 구조와 규칙을 명확히 표현하는 기법
물리적 데이터모델링: 논리 데이터 모델이 데이터저장소로서 어떻게 컴퓨터 하드웨어에 표현될 것인가를 다룸
노드 인증: 관리 서버와 데이터 콜렉터가 보안 방식으로 서로 통신하는지 확인하는 기술
DR (Disaster Recovery, 재해 복구): 중단 발생 후 가용한 툴과 기술을 이용해 IT 서비스를 복구하는 일련의 단계
BC (Business Continuity, 비즈니스 연속성,회복성): 광범위한 형태의 데이터 보호. 복원을 포함해 비즈니스 운영을 지속하기 위한 절차 포함
CDP(Continuous Data Protection, 지속적 데이터 보호): 모든 데이터 변경본을 자동 복사해 관리자가 언제든 데이터를 복원할 수 있게 끔하는 데이터 백업
HA(High Availability, 고가용성): 재해 중 비즈니스를 지속하는데 도움이 될 수 있는 기술 특성. 손상 시 failover하여 원래 시스템 대체. 보호 기능은 아님
Repllication(복제): 재해 시 최신 데이터를 복원할 수 있게, 다른 위치로 데이터를 복사하는 프로세스
오픈뱅킹: 다른 금융기관이나 핀테크 서비스임에도 내 금융정보를 조회하거나 이체하는 등의 서비스를 제공해주는 것
마이데이터: 공공기관이나 금융권에 분산된 고객신용정보를 하나의 플랫폼에서 통합 관리하도록 제공해주는 서비스.
	소비 습관, 자산 현황 등을 종합적으로 분석해서 유익한 금융상품을 추천해줌
고도화 프로젝트: 기존에 사용하던 SW, HW를 가지고 기능 또는 제품 보강 및 개선하는 프로젝트
차세대 프로젝트: 기존에 사용하던 SW, HW 환경을 탈바꿈하여 기능 또는 제품을 새롭게 바꾸는 프로젝트
RFP (Request For Proposal): 클라이언트가 업체에 요구한 사항 / 프로젝트 목적, 범위, 예산, 일정, 기술적 요구사항 등
Scope (범위): 프로젝트에서 수행될 구체적인 작업 / 무엇부터 어느 지점까지 포함하는가의 범위
MileStone: 프로젝트 일정에서 중요한 지점이나 목표
KickOff: 프로젝트의 공식적인 시작
WBS (Work Breakdown Structure, 업무 분류 체계): 프로젝트를 더 작고 관리하기 쉬운 작업들로 나누는 방법 / 팀, 부서 역할 명확화
API (Application Programming Interface): 소프트웨어 시스템간 연결하는 방식
UAT (User Acceptance Testing): 고객이 직접 시스템을 테스트하여 요구사항을 충족하는지 확인하는 단계
회귀테스트 (Regression Test): 새로운 기능 추가나 버그 수정후, 기존 기능들이 여전히 제대로 작동하는지 확인하는 테스트
Go-Live: 공식적으로 런칭되어 실제 사용되기 시작하는 시점
SLA (Service Level Agreement): 업체가 시스템을 어떻게 지원할 것인지 정의한 계약
Patch: 문제를 해결하거나 소소한 기능을 추가하기 위한 비교적 작은 업데이트 / 보안 강화, 버그 수정, 성능 개선 등을 위해 주기적으로 적용
Scalability (확장성): SI 시스템이 사용자나 데이터를 처리할 수 있는 능력
무결성: 데이터 값이 정확한 상태
정합성: 데이터의 정확 유무와 별개로 데이터들의 값이 서로 일치하는 상태
클라우드 컴퓨팅: 인터넷을 통해 컴퓨팅 자원을 필요에 따라 사용할 수 있게 해주는 서비스 / 서버, 저장소, DB 등 포함
NAC (Network Access Control): 엔드포인트 보안기술 / 사용자 또는 시스템 인증 및 네트워크 보안을 통합하는 보안 접근 방식
NAT (Network Address Translation): 사설 IP를 공인 IP로 변환하는 서비스
NAS (Network Attached Storage): 네트워크 환경 데이터 스토리지
DMA (Direct Market Access): 증권사 서버를 거치지 않고 고객매매시스템을 바로 최소주문경로(최소속도)로 거래소 시스템에 연결하는 주문방법
algorithmic trading: 하나 혹은 여러개의 지표에서 신호가 발생하면 그에 따라서 기계적으로 프로그래밍된 상태로 주문을 실행하는 트레이딩기법
ELW (Equity Linked Warranty): 특정 대상물(기초자산)을 사전에 정한 미래의 시기(만기일 혹은 행사기간)에 미리 정한 가격에 사거나 팔 권리를 갖는 유가증권
HTF (High Frequency Trading): 컴퓨터를 통해 빠르게 주문을 수많이 반복하는 거래. 알고리즘매매방식
Latency: 원하는 결과를 만들어 내기까지 발생하는 지연시간의 총합 / low latency: 호가나 주문을 경쟁자보다 빨리 받거나 보냄
System Trading: 자의적 판단이나 편견을 배제하고 일정한 매매규칙을 사용해 일관성 있게 매매를 수행함으로써 투자수익률을 높이는 매매방법
	특정종목의 매수,매도 가격 등 조건을 프로그래밍화해 놓으면 컴퓨터가 조건대로 매매해주는 시스템
Co-location Service: 기업이나 사용자가 서버를 구매해서 서버업체에 위탁,관리를 의뢰하여 서버, 회선비만 지불하고 서버를 관리하는 서비스
ATS (Alternative Trade System): 대체거래소
Brokerage Fee: 중개 상인이나 증권업자가 고객의 위탁을 받아 물건이나 증권을 매매하여 주는 대가로 받는 돈
Buy-side: 헤지펀드, 자산운용사 등
Sell-sie: 투자중개업자
PB (PowerBase): 파생상품, 자산관리, 채권딜링, 투자 정보, 글로벌 트레이딩 등 모든 IT업무를 지원하는 토탈 아웃소싱 서비스
RDMA (Remote Direct Memory Access): 물리적으로 다른 여러대 서버의 메모리를 가상화시켜 하나의 거대한 메모리 풀로 만들어주는 기술
	메모리와 서버가 직접 통신하기 때문에 DB, 가상머신 마이그레이션, DW, 분석 등 성능을 대폭 향상
MDD ( Model-Driven Development): SW 개발 시 모델을 작성하면 프로그램 코드뿐 아니라 다양한 문서를 자동으로 생성하는 개발 방식
OSI 7계층: 네트워크에서 통신이 일어나는 과정을 7단계로 나눈 것
1. 물리 계층(Physical Layer): 하드웨어 시스템으로 구현(2계층부터는 SW적으로 구현) / 전기적, 기계적, 기능적 특성을 이용해 통신 게이블로 데이터 전달
	사용되는 통신 단위는 Bit(1또는0) / 데이터 전달만 하며 데이터가 무엇인지, 어떤 상태인지 신경쓰지 않음 / 통신 케이블, 리피터, 허브
2. 데이터링크 계층(DataLink Layer): 물리계층을 통해 송수신 되는 정보의 오류와 흐름을 관리하여 정보 전달을 수행. 오류 검출, 재전송, 흐름제어
	맥 주소를 가지고 통신. 전송되는 데이터 단위를 프레임 / 브리지, 스위치
3. 네트워크 계층(Network Layer): 주소(IP)를 정하고 설정한 경로(Route)에 따라 패킷을 전달 / 데이터를 목적지까지 가장 안전하고 빠르게 전달(라우팅)
	전달 경로 선택은 미리 정해지는 정적(Static), 현재 부하 상태에 따라 결정되는 동적(Dynamic) 방식으로 구분 /전송 데이터를 패킷(Packet)
4. 전송 계층(Transport Layer): 통신 활성화 계층 / TCP프로토콜을 이용, 포트를 열어 응용프로그램들이 전송할 수 있게 함 
	TCP: 신뢰성, 연결 지향적 / UDP: 비신뢰성, 비연결성, 실시간 / 받은 데이터를 합쳐 5계층에 보냄
	송신, 수신 프로세스를 직접 연결하는 단대단(End-to-End) 통신 기능. 패킷의 전송이 유효한지 확인하고, 전송 실패된 패킷을 다시 전송
5. 세션 계층(Session Layer): 데이터가 통신하기 위한 논리적인 연결(대문 역할) / 세션 설정, 유지, 종료, 전송 중단 시 복구 기능
	전송 계층과 유사하나, 사용자에게 원격 파일 전송이나 원격 로그인 같은 상위적 연결 개념인 세션 기능을 제공
	송수신 호스트 사이의 대하 제어를 비롯해 상호 배타적인 동작을 제어하기 위한 토근 제어, 일시적 전송 장애를 해결하기 위한 동기(Synchronization) 기능 제공
6. 표현 계층(Presentation Layer): 데이터의 의미(Semantic), 표현방법(Syntax)를 처리. 데이터의 표현 방법이 다르면 서로 데이터를 이해할 수 있게 변환
	데이터 암호화, 인코딩, 명령어 포장, 압축 등 / JPEG, MPEG, GIF, ASCII 등
7. 응용 계층(Application Layer): 최종 목적지. 응용 프로세스와 직접 관계하여 일반적인 응용 서비스 수행(explore, chrome 등)
	허가된 사용자가 적절한 접근 권한에 따라 사용할 수 있어야 함 / HTTP, FTP(File Transfer Protocol), SMTP, Telnet 
그리드 컴퓨팅: WAN (Wide Area Network, 원거리 통신망)으로 연결된 서로 다른 기종의 컴퓨터들을 묶어 가상의 대용량 고성능 컴퓨터를 구성하여
	고도의 연산 작업 혹은 대용량 처리를 수행하는 것
TCP/IP 4계층
1. 네트워크 액세스 계층(Network Access Layer) = 물리(1)+데이터링크 계층(2) / TCP/IP패킷을 네트워크 매체와 송수하는 과정 담당
에러 검출, 패킷의 프레임화 / 네트워크 접근방법, 프레임 포맷, 매체에 대해 독립적 동작하게 설계 /물리적 주소로 MAC 사용 / LAN, 패킷망 등에 사용됨
2. 인터넷 계층(Internet Layer) = 네트워크 계층(3) / 주소지정, 패키징, 라우팅 기능 / 네트워크상 최종 목적지까지 정확히 연결하는 기능 / IP, ARP, RARP
3. 전송 계층(Transport Layer) = 전송 계층(4) / IP와 Port를 이용해 프로세스와 통신 / 응용계층의 세션과 데이터그램 통신 서비스 제공
동신 노드간 연결 제어, 신뢰성 있는 데이터 전송 담당 / TCP, UDP
4. 응용 계층(Application Layer) = 세션(5)+표현(6)+응용계층(7) / 프로그램(브라우저)가 직접 수행하는 레이어, 데이터를 처음으로 받는 곳
다른 계층의 서비스에 접근할 수 있게 하는 앱 제공 / 앱들이 데이터를 교환하기 위해 사용하는 프로토콜 정의
