## 금융
책무: 금융회사 또는 금융회사 임직원이 준수해야하는 사항에 대한 내부통제 및 위험관리의 집행 및 운영에 따른 책임
	1. 지정 책임자가 총괄적으로 수행하는 책무(준법감시인, 위험관리책임자 등)
		- 위험관리업무와 관련된 책무 / 준법감시업무와 관련된 책무 / 정보보안 업무와 관련된 책무
	2. 금융영업 관련 책무(인허가 받은 고유 업무와 부수, 겸영업무)
		- 예금 및 적금업무와 관련된 책무 / 투자매매업무와 관련된 책무 / 신탁업무와 관련된 책무 / 건전성 관리업무와 관련된 책무
	3. 경영관리 관련 책무(영업 업무 영위를 위해 수행하는)
		- 인사업무와 관련된 책무, 광고업무와 관련된 책무, 자회사 관리업무와 관련된 책무, 건전성 관리업무와 관련된 책무
책무구조도(24.07.03시행): 임원의 직책별로 책무를 배분한 문서
	구성) 책무체계도: 책무체계를 일괄적으로 파악할 수 있는 도표 / 책무기술서: 임원 별 책무의 상세내용을 기술한 문서
감경, 면제: 관리,감독의 책임이 있는 사람이 상당한 주의를 다한 경우 / 금융위 "내부통제관리의무 위반 관련 제재 운영지침(안)"
	-위법행위 등 결과발생에 대한 예측이 가능하였는지 여부 / 실효성 있는 관리조치 이행 여부
자점감사: 각 지점에서 처리한 업무가 규정이나 지침을 지켰는지 여부를 자체적으로 점검하는 일
전문: 프로그램들이 서로 데이터를 주고 받는 "양식"을 고정된 형태로 저장해 놓는 것
채널계: 사용자가 접속하는 다양한 채널의 데이터를 관리하는 시스템 / 모바일 뱅킹, 인터넷 뱅킹, ATM, 콜센터, 현업 단말기 등 / DMZ / PT
	계정계의 데이터를 전문을 통해 화면에 출력, 사용자(End User)의 요청을 계정계에 전달하고, 처리결과를 노출
계정계: 고객 거래 데이터, 계좌, 계정을 처리하는 핵심 시스템 / 공통, 수신, 여신, 신탁, 보험, 카드, 외환, 대행업무 등으로 구성 / BT
	신탁: 고객(위탁자)이 금전 또는 금전외 재산(유가증권, 부동산, 금전채권 등)의 재산권을 증권(수탁자)에 이전 또는 처분하며 운용지시
	-> 수탁자는 위탁자가 지정한 수익자(위타자 또는 위탁자가 지정한 제3자)의 이익 또는 특정 목적을 위해 이전된 재산권을 관리, 처분하는 제도
정보계: 계정계에서 받은 데이터를 바탕으로 거래 활동 및 성과를 측정하고 분석하는 시스템 / DT
	데이터 접근 속도 중요 / 수익 관리, 고객관계관리, 성과관리, 위험관리 등
운영계: 서비스 운영을 위한 유지보수, 통합관제, 모니터링등의 시스템 / IT인프라(HW, SW), 네트워크, 보안 영역 등
대외계: 금융기관이 외부기관과 연계되는 시스템 / 은행 공동망, 제휴 기관, 금융 결제원 등 / 안정성과 정확성 중요 / DMZ
AS-IS: 있는 그대로, 현재 업무 프로세스 분석
TO-BE: 미래의, 개선될 업무 프로세스에 대한 분석
PT(Presentation Tier): 채널계, 사용자가 입력한 정보를 BT에 전달
BT(Business Tier): 계정계, 비지니스 로직 영역. 데이터 처리, 비지니스 규칙 포함
DT(Data Tier): 정보계, 데이터 저장, 관리, 처리
고유업무: 저축을 받아 돈을 빌려주는 것 / 은행 본연의 임무 / 수신(저축), 여신(대출), 외환
부수업무: 고유업무를 하며 동반되는 업무 / 보증, 어음 인수
겸영업무: 영업을 겸함 / 채권회수(추심) 대행, 기업 M&A 중개
오픈뱅킹: 다른 금융기관(핀테크) 서비스임에도 내 금융정보를 조회하거나 이체하는 등의 서비스를 제공해주는 것 / 하나의 앱으로 여러 금융기관 서비스 이용
마이데이터: 각종 공공기관이나 금융권에 분산된 고객정보를 하나의 플랫폼에서 통합 관리해주는 서비스 / 소비 습관, 자산 현황 등을 종합 분석해 금융상품 추천
미들웨어: 양 쪽을 연결해 데이터를 주고 받을 수 있도록 매개 역할을 하는 소프트웨어 / TP 모니터 (Transaction Processing Monitor)
MCA (Multi Channel Architecture): 기업 내부 동기종 또는 유사기종 시스템 연계(은행에서의 여신-수신 업무 연계 등)
SOA (Service Oriented Architecture, 서비스 지향 아키텍쳐): 기존 앱의 기능들을 비즈니스적 의미를 가지는 기능 단위로 묶어 
	컴포넌트 단위로 재조합 후, 업무기능 구현 앱을 만드는 소프트웨어 아키택처
- 모듈연계(ESB, EAI): 내부 모듈과 외부 또는 내부 모듈 간 데이터 교환을 위해 관계를 설정하는 것
ESB (Enterprise Service Bus): 기업에서 여러 서비스들을 서로 연결해 상호 조합, 연계, 운용하는 인프라 시스템 / 분산된 컴포넌트를 통합하여 연동
	프로세스 통합 / 표준기술 사용(Web Service, XML) / 기업 내 외부 통합 / 서비스 중심으로 프로세스 진행
EAI (Enterprise Application Integration, 기업 앱 통합): 기업 내부 이기종 시스템을 연계하여 통합하는 솔루션 / EAI에 접근 시 어댑터를 거침
	애플리케이션 통합 / 벤더 종속적 기술 사용 / 중앙집중식 / 기업 내 통합 / 시스템 사이에서 시스템 연계중심 / 개별 애플리케이션에서 수행
	[Hub & Spoke]: 시스템 간 1:1이 아니라, 중앙에 EAI가 허브 역할을 하면서 모든 통신이 EAI를 거치게 함
FEP (Front End Processor): 메인프레임에서 통신 과부하를 줄이기 위해 전처리 작업하는 과정. / 금융권에서는 B2B 연계
ERP (Enterprise Resource Planning, 전사적 자원 관리): 기업 내부의 모든 인적, 물적 자원을 효율적으로 관리할 수 있는 기업의 통합 정보 시스템
CRM (Customer Relationship Management, 고객 관계 관리): 고객 정보를 수집, 분석해 고객의 성향에 맞는 관계를 구축하는 것
SCM (Supply Chain Management, 공급망 관리): 유통 생산에 관련된 과정을 관리하는 것
방카슈랑스 (은행+Assurance(보험)): 은행 등 금융기관이 보험회사의 대리점과 같은 역할을 수행하여 보험 상품을 판매하는 것
FAMS: 통합 전자거래 중계시스템
AML (Anti-Money Laundering, 자금세탁 차단 관리 체계)
ITSM (IT Service Mangement): 정보기술 서비스들을 계획, 설계, 운영하기 위해 수행되는 활동
BPR (Business Process Reengineering, 업무 재설계): 기업의 활동이나 업무의 전반적 흐름을 분석하고, 경영 목표에 맞게 조직과 사업을 최적으로 다시 설계해 구성하는 것
RM (Relationship Manager):  기업고객에 대한 여신지원 결정, 정보제공, 경영상담 등 금융 솔루션 제공하는 방식
ISP (Information Strategy Planning, 정보전략계획): 조직의 경영계획과 목표를 지원하기 위한 정보시스템의 비전 계획
SM (System Management, 시스템 관리): 대기업 혹은 공공기관에서 제공해주는 장소에서 이미 개발된 시스템의 유지보수를 맡거나 추가 개발하는 것
ECM (Enterprise Content Management: 전사적 콘텐츠 관리): 조직 내 처리 업무에 관한 콘텐츠나 문서를 보관, 전달, 관리에 이용하는 기술. 비구조화 정보가 어디있든, 관리할 수 있는 형태의 환경으로 만듦
	WCMS: Web Content Management System, 웹 콘텐츠 관리 시스템: 비전문가도 웹 콘텐츠를 효과적으로 관리해주는 솔루션
	DAM: Digital Asset Management, 디지털 자산관리: 디지털 자산을 생성, 저장, 검색하고 사용권한을 관리하기 위한 비즈니스 시스템
CSR (Corporate Social Responsibility, 기업의 사회적 책임): 기업의 이윤추구활동 이외에 법령과 윤리를 준수하고,
	기업의 이해관계자의 요구에 적절히 대응함으로써 사회에 긍정적 영향을 미치는 책임있는 활동
CF(Cash Flow) : 현금흐름
COA(Chart Of Account) : 계정과목일람표
CoC(Cost Of Capital) : 자본비용. 자본을 사용하기 위해 지불해야 하는 대가로, IFRS17에서 위험조정을 계산하는 방법중 하나. 자본비용율에 요구자본비용을 곱하여 위험조정값을 계산함.
CSM(Contractual Service Margin) : 보험계약마진. 보험계약집합에 대한 자산 또는 부채장금액의 구성요소러소, 집합 내 보험계약에 따라 서비스를 제공하면서 인식하게 될 미실현이익을 나타냄.
CSM이 +면 이익, -면 손실
ARK(Agile, Reliable, Keen) : 보험개발원에서 공동개발 중인 부채평가시스템.
BEL(Building Block Approach) : 최적추정(가정) 부채.
G/L(General Ledger) : 총계정원장. 기업 회계상 모든 계정의 수입과 지출 등을 수록한 장부
GOC(Group Of Contracts) : 계약그룹(집합)
IFRS(International Financial Reporting Standards) : 국제회계기준. 기업의 회계 처리와 재무제표에 대한 국제적 통일성을 높이기 위해 국제회계기준위원회에서 마련해 공표하는 회계기준.
K_ICS(Korea-Insurance Capital Standard) : 신지급여력제도. 새로운 보험사 지급여력제도로 2021년 도입되는 IFRS17에 따라 기존 원가평가방식에서 시가평가방식으로 재무건전성을 평가하는 제도.
K-IFRS(KOREA - International Financial Reporting Standards) : 한국채택국제회계기준. 계정과목을 세세하게 구분하지 않고 포괄적으로 기재함.
LAT(Liability Adequacy Test) : 보험부채적정성평가. 보험계약으로부터 발생할 미래 현금유입과 유출액을 현재 가치로 바꿔 책임준비금이 적정한 지를 따지는 제도.
RA(Risk Adjustment) : 위험관리
SAP(Statutory Accounting Principles) : 감독회계기준
hierarchical: 계층적
ITO : 전산용역비
CSR :  Client Server Request 인듯
채널 : 상품을 파는 경로.
TM : Tele Marketing. 전화로 하는 영업. I/B는 인바운드(고객->콜센터) O/B는 아웃바운드(콜센터->고객)
GA : 대리점. 회사에서 직접 운영하는것은 지점 이라 하고, 흥국에서 몇개, 삼성에서 몇개, 한화에서 몇개 이런식으로 다양한 회사의 상품을 운영하는데를 대리점이라고 한다.
AD( Activity Driver) :  활동원가동인
RD(Resource Driver) : 전표원가동인?
약대 : 약관대출
수보 : 수입보험료
유지율 : 해지율의 반대
전사 : 회사 전체
공통비 : 한 영역에서만 쓰지않고 공통적으로 쓴 비용.
판관비 : 판매비와 관리비. 매출액 = 매출원가 + 판관비
증번 : 증권번호.
배부 : Based Costing
이수관 : 관리점 변경
이연 : 특정한 사정으로 인해 손익의 귀속이 다음 회기로 연장되는 것. 
예상 : 이연의 반댓말. 내년에 귀속되어야 할 손익이 올해의 손익으로 처리되는 것.
감가상각 : 시간의 흐름에 따른 자산의 가치 감소(감가)를 회계에 반영하는 것(상각)
선수금 : 회사에서 제품이나 용역을 제공하기 전에 미리 받은 금액. 부채이지만 현금 유동성이 좋아진다.
당기순이익: 영업이익에서 영업외비용(이자수익, 배당금, 임대료 수익, 손실 등) 가감 후, 법인세까지 차감한 최종적 수입
DMA (Direct Market Access): 증권사 서버를 거치지 않고 고객매매시스템을 바로 최소주문경로(최소속도)로 거래소 시스템에 연결하는 주문방법
algorithmic trading: 하나 혹은 여러개의 지표에서 신호가 발생하면 그에 따라서 기계적으로 프로그래밍된 상태로 주문을 실행하는 트레이딩기법
ELW (Equity Linked Warranty): 특정 대상물(기초자산)을 사전에 정한 미래의 시기(만기일 혹은 행사기간)에 미리 정한 가격에 사거나 팔 권리를 갖는 유가증권
HFT (High Frequency Trading): 컴퓨터를 통해 빠르게 주문을 반복하는 알고리즘매매방식
Latency: 원하는 결과를 만들어 내기까지 발생하는 지연시간의 총합 / low latency: 호가나 주문을 경쟁자보다 빨리 받거나 보냄
System Trading: 자의적 판단이나 편견을 배제하고 일정한 매매규칙을 사용해 일관성 있게 매매를 수행함으로써 투자수익률을 높이는 매매방법
	특정종목의 매수,매도 가격 등 조건을 프로그래밍화해 놓으면 컴퓨터가 조건대로 매매해주는 시스템
ATS (Alternative Trade System): 대체거래소
Brokerage Fee: 중개 상인이나 증권업자가 고객의 위탁을 받아 물건이나 증권을 매매하여 주는 대가로 받는 돈
Buy-side: 헤지펀드, 자산운용사 등
Sell-side: 투자중개업자
PB (Private Banking): 개인 고객의 자금을 운용할 수 있게 관리하는 영업방식
PB (PowerBase): 파생상품, 자산관리, 채권딜링, 투자 정보, 글로벌 트레이딩 등 모든 IT업무를 지원하는 토탈 아웃소싱 서비스
고도화 프로젝트: 기존에 사용하던 SW, HW를 가지고 기능 또는 제품 보강 및 개선하는 프로젝트
차세대 프로젝트: 기존에 사용하던 SW, HW 환경을 탈바꿈하여 기능 또는 제품을 새롭게 바꾸는 프로젝트


## IT
ODS (Operational Data Store, 운영 데이터 저장소): 데이터를 추출하고 처리하기 위해 최종 스토리지로 전송하기 전, 데이터 처리를 위한 임시 스토리지
MSA (Micro Service Architecture)
CDC솔루션( Change Data Capture): 마지막 추출 후 변경된 데이터만 골라내는 기술. 시스템 로드를 줄이고 작업 생산성 향상 / 주로 DB 로그 기반 CDC
Redhat: 리눅스
Ontune: 팀스톤 / 시스템 성능 관제 솔루션
CLIP Report: 클립소프트 / 데이터(RDB, XML, CSV, JSON, SAP 등)를 통계 양식, 증명서 등으로 변환하여 조회 및 출력할 수 있게 하는 솔루션
Weblogic: 오라클 / WAS / 트랜잭션 처리, 메시징 서비스, 디렉토리 서비스 등 톰캣이 처리하지 못하는 기능 지원
iDesigner: IBK시스템 / iFramework의 웹화면 개발 도구
IBSheet: 웹 환경에서 데이터를 관리하고 모니터링하는 프로그램 / 대용량 데이터 처리, 스르킵트 언어 호환
Sybase ASE: RDBMS (관계형 데이터베이스 관리 프로그램)
Fleta-SRM: 스토리지 자원관리 솔루션
TSM Agent: Tableau 서버 관리 프로그램
Nexacro: 투비소프트 / 프론트엔드 개발용 프레임워크(IDE)
Control-M: 이기종 서버 및 앱 환경에서의 통합된 작업관리를 제공하는 솔루션
한글로폼: 워드프로세스 기반 문서를 DB와 연계하여 처리하고, 스타일을 자동화해주는 솔루션
Tibero EE (Enterprise Edition): 데이터베이스(오라클류)
Jennifer: 웹 앱(Java EE 등) 시스템 모니터링을 위한 APM (Application Performance Monitoring) 솔루션
Petra Cipher: 중요 데이터의 손실을 방지하는 DB 암호화 솔루션
MaxGauge: DB들을 통합적으로 관리할 수 있는 모니터링 솔루션
tomcat: (WAS) 자바 servlet을 실행시키고, JSP 코드가 포함된 동적 웹 페이지를 구동시켜주는 프로그램
NoSQL: 비관계형 DB / 대량의 분산된 비정형 데이터 특화 / Join 불가 / 대용량 데이터 저장 / 분산형 구조 / 고정되지 않은 테이블 스키마
IaaS (서비스형 인프라): 자원만 주는 것 / AWS EC2 네이버 클라우드 등
PaaS (서비스형 플랫폼): 플랫폼을 가상화해서 웹으로 제공하는 형태(가상 인프라를 통해 미들웨어, 런타임, OS가 포함)
SaaS (Software as a Service, 서비스형 소프트웨어): 설치하지 않고 온라인으로 실행 가능한 소프트웨어
		 / 오피스365, 구글 독스, 인사관리 소프트웨어, 전자 상거래 시스템, 고객 관리 툴, 통합개발환경 서비스 등
AES (Advanced Encryption Standard, 대칭키 암호화): 암호화키 = 복호화키
	데이터 전송, 파일 암호화, VPN 등 대량 데이터 암호화 / HTTPS를 통해 전송되는 데이터 암호화, Wi-Fi WPA2 보안 프로토콜 등
RSA (Rivest-Shamir-Adleman, 공개키 암호화): 암호화키=공개키, 복호화키=개인키
	작은 데이터의 안전한 전송, 키 교화, 인증서 서명, SSL/TLS 인증서에서 클라-서버 간 키 교환
	- 공개키로 암호화된 데이터는 비밀키로만 복호화 가능
	- 비밀키로 서명된 데이터는 공개키로만 검증 가능
	- 공개키: 일반적으로 서버가 소유 -> 클라이언트에게 배포
		-암호화: 서버로 데이터를 보낼 때 공개키로 암호화 (서버의 개인키 없이 복호화X)
		-인증: 클라이언트는 서버가 제공하는 공개키를 사용해 서버의 신원 확인, 공개키가 인증서와 함께 제공, 이 인증서를 통해 서버의 신뢰성 검증
	- 개인키: 서버가 소유, 외부에 공개X, 암호화된 데이터를 복호화하고, 서버의 신원을 증명
		-복호화: 클라이언트가 서버의 공개키로 암호화한 데이터를 서버는 자신의 개인키로 복호화할 수 있음
		-디지털 서명: 서버는 개인키로 데이터를 서명하여, 클라이언트에게 자신이 신뢰할 수 있는 서버임을 증명
End Point: API가 서버에서 리소스에 접근할 수 있게 하는 URL
	서비스의 실제 비즈니스를 수행하기 위해 연결하는 연결고리. 바인딩할 경우 클라이언트에서 웹 서비스 호출 가능
정적(Static)페이지: 미리 작성된 내용이 고정되어 있고, 요청에 따라 동일한 내용 제공 / HTML, CSS, JavaScript, IMG / 웹서버에 의해 처리
동적(Dynamic)페이지: 요청에 따라 서버에서 동적으로 생성되는 페이지 / DB조회, 사용자 입력 처리, 외부API 호출 / WAS에 의해 처리
Web서버: 클라이언트로부터 HTTP 요청을 받아 응답 / WAS를 거치지 않으면 정적페이지(HTML, CSS, IMG) / WAS에게 요청 전달
	정적 컨텐츠가 있는 경로만 명시하고, 실제로는 NAS(Network Attached Storage)에 있음
WAS(Web Application Server) : DB상호작용, 로직처리, 동적페이지, 세션 및 상태 유지 기능 / HTTP를 통해 컴퓨터나 장치에 애플리케이션을 수행
nginx: web서버, 로드밸런서, http 캐시 서버로 사용되는 오픈소스 소프트웨어
node.js: 서버 사이드에서 js를 실행할 수 있는 js 런타임(특정 언어가 구동되는 환경)
Proxy(대리): 서버와 서버 사이에서, 서버를 대신해 작업을 수행해주는 중개인 / 요청이 본 서버에 도착 전 프록시 서버에서 중복 요청을 처리(주로 Forward proxy)
	가정: 프록시가 없는 경우) localhost:3000 웹서버 오픈시 사용자가 갑자기 많아지거나, 웹서버가 그대로 노출되면 보안 위험
				nginx 사용시 로드밸런싱, 분산처리, 웹서버의 SSL인증도 가능
Forward Proxy: 클라이언트 바로 뒤에 위치해 클라이언트의 요청을 대신 처리 / VPN, 클라이언트의 정보 암호화, 캐싱(반복 요청에 대한 응답을 서버에 보관해두고 즉각 응답)
Reverse Proxy: 웹서버나 WAS 바로 앞에 위치해 서버의 요청을 대신 처리 / 로드밸런싱, 서버 정보 암호화/ 서버를 WEB, WAS로 나누면 WEB서버가 프록시
batch: 사용자의 개입없이 실행을 스케쥴링할 수 있는 작업. 프로그램 흐름에 따라 순차적으로 자료를 처리하는 방식
batch process : 일괄 처리. 주기적으로 대량의 반복적인 데이터 작업을 완료하기 위해 사용하는 방식 / 보험회사 - 우편번호 가져오기 예시
	- 비 대화형 처리: 사용자와 상호작용X
	- 대량 데이터 처리
	- 스케쥴링: 낮은 트래픽 시간대에 실행하여 시스템 부담 최소화
	- 오류 처리 및 로그: 수행 작업에 대한 로그
QA(Quality Assurance) : 품질보증. 프로젝트가 관련된 품질기준을 충족시킨다는 것을 확신시키기 위하여, 정기적으로 프로젝트의 전반적인 수행을 평가하는 절차.
PMO(Project Management Office) : 프로젝트 관리업무 전담 조직
R&R(Role and Responsibilities) : 프로젝트 내에서의 프로세스 및 조직의 구성원들이 수행해야할 역할과 그 역할의 수행에 따른 책임관계를 분명히 하는 것
DR (Disaster Recovery, 재해 복구): 중단 발생 후 가용한 툴과 기술을 이용해 IT 서비스를 복구하는 일련의 단계
BC (Business Continuity, 비즈니스 연속성,회복성): 광범위한 형태의 데이터 보호. 복원을 포함해 비즈니스 운영을 지속하기 위한 절차 포함
CDP(Continuous Data Protection, 지속적 데이터 보호): 모든 데이터 변경본을 자동 복사해 관리자가 언제든 데이터를 복원할 수 있게 끔하는 데이터 백업
HA(High Availability, 고가용성): 재해 중 비즈니스를 지속하는데 도움이 될 수 있는 기술 특성. 손상 시 failover하여 원래 시스템 대체. 보호 기능은 아님
RFP (Request For Proposal): 클라이언트가 업체에 요구한 사항 / 프로젝트 목적, 범위, 예산, 일정, 기술적 요구사항 등
Scope (범위): 프로젝트에서 수행될 구체적인 작업 / 무엇부터 어느 지점까지 포함하는가의 범위
MileStone: 프로젝트 일정에서 중요한 지점이나 목표
KickOff: 프로젝트의 공식적인 시작
WBS (Work Breakdown Structure, 업무 분류 체계): 프로젝트를 더 작고 관리하기 쉬운 작업들로 나누는 방법 / 팀, 부서 역할 명확화
NAS (Network Attached Storage): 네트워크 환경 데이터 스토리지




## 컴퓨터공학
네이티브 앱: 앱의 기능을 중점적으로 사용 / 파워포인트(pc), 스토어의 대부분의 어플 / OS 종속, 비용 높음
웹 앱: 앱처럼 생긴 웹 페이지 / 설치하지 않으므로 디바이스의 자체 기능 사용 불가, 비용 낮음(웹 개발 비슷)
	SPA(단일 페이지 방식): 최초 한 번 페이지 전체를 받아오고, 화면 변경, 요청 시 해당 부분만 Ajax를 통해 데이터를 출력
하이브리드 앱: 구글메일, 인스타그램, 트위터... / 디바이스 기능 사용 가능
가용성(Availability): 서버와 네트워크, 프로그램 등의 정보 시스템이 정상적으로 사용 가능한 정도 = 가동률
개인정보 비식별화: 정보 집합물에서 식별정보를 제거해 개인을 특정할 수 없게 가명화, 익명화하는 방법과 조치
데이터 3법: 개인정보 보호법, 정보통신망법, 신용정보 보호법
	개인정보(특정 가능) - 구체적인 동의 후 활용 가능 / 가명 정보 - 동의 없이 통계, 연구, 공익적 기록 등 가능 / 익명정보 - 자유
노드 인증: 관리 서버와 데이터 콜렉터가 보안 방식으로 서로 통신하는지 확인하기 위해 사용되는 기술
부인방지: 메시지의 송수신이나 교환 후, 또는 통신이나 처리 실행 후 그 사실을 사후에 증명함으로써 사실 부인을 방지하는 보안 기술.
라우팅: 네트워크 안에서 통신 데이터를 보낼 경로를 선택하는 과정 / 패킷 스위칭 네트워크에서 패킷 전달 및 라우터,브릿지,게이트웨이,방화벽,
	스위치라 불리는 중간 노드를 거쳐 논리적으로 주소가 부여된 패킷의 변환을 총괄
망 분리: 외부 공격으로부터 내부 자료 보호를 위해 업무용 내부 망과 인터넷 망을 분리하는 것
락인(Rock-in): 다른 제품으로 소비 전환하지 않고 기존 서비스를 꾸준히 유지하는 현상 / 사용자 고착, 자물쇠 효과
블록체인: 네트워크 내 모든 참여자가 공동으로 거래 정보를 검증,기록,보관함으로써 '공인된 제3자'없이 거래 기록의 신뢰성을 확보하는 기술.
	특정시간(10분) 발생된 모든 거래정보가 기록된 Block을 생성, 모든 구성원에게 전송해서 같은 거래장부 사본을 나눠 보관하는 분산원장 기술
	공개키 암호화 기술과 작업증명(Proof-of-Work) 매커니즘을 사용해 거래 기록의 신뢰성을 확보
SHA: 미국 NSA가 제작하고 미국 국립표준기술연구소(NIST)에서 표준으로 제작한 해시 암호 알고리즘 / 인터넷뱅킹 등
On Demand Batch: 배치작업을 스케쥴링이 아닌, 사용자 요청시에만 하는 것
결함 : 통합테스트를 했을때 프로젝트가 요구사항이나 사양을 충족하지 못하여 수정 또는 교체가 필요한 불완전한 상태
프리징 : 프로젝트/시스템에 영향을 주는 변경사항에 대해 더이상 수용하지 않는 시점을 의미함. 프리징 기간에는 로컬에서만 변동하고 서버로 업데이트 하지않는다.
이슈 : 분쟁의 대상이 되는 쟁점 또는 해결되지 않고 논의중에 있거나 상반되는 견해를 가진 문제.
DATA MART : 데이터 저장소. 특정 사용자가 관심을 갖는 데이터들을 담은 비교적 작은 규모의 데이터 웨어하우스의 형태.
ERD(Entity Relationship Dagram) : 개체-관계 다이억그램. 어플리케이션 개발자 간의 자료를 공통적으로 이해할 수 있도록 하는 매체가 됨.
ETL(Extraction, Transformation, Loading) : 데이터의 추출, 변환, 적재를 일컫는 말.
SI(System Integration) : 시스템을 구성하는 기기, 소프트웨어 및 관련하는 기술을 선정하고 또한 정비하여 정보처리시스템으로서 종합적으로 구축하는 행위
PRA (Robotic Process Automation, 로보틱 처리 자동화): 사람이 반복적으로 처리하는 업무를 로봇 소프트웨어를 통해 자동화 하는 솔루션
	정보통신 시스템간 손쉬운 연결을 통해 직원들이 반복적인 작업에서 벗어나, 창의적인 업무 수행이 가능, 인력 감축
U2L: 주 전산시스템을 유닉스에서 리눅스로 전환하는 것
오픈 아키텍처: 플랫폼 상 기업과 고객, 고객과 고객을 연결해주기 위해 필요한 기반인 시스템.
	기업의 시스템 또는 콘텐츠를 개방하고 공유함으로써 외부의 개발자 및 사용자들이 새로운 창작물을 만들거나 문제를 해결할 수 있음
오픈소스 SW (OSS): SW개발자의 권리를 보장하면서, 소스코드를 누구나 열람할 수 있게 하는 공개된 개발 모델.
	 소스코드 사용,복제,배포,수정 허용. 사용자에게도 라이선스를 줌으로써 권리와 의무를 부여
도메인 네임: 네트워크 상에서 컴퓨터를 식별하는 호스트명
데이터 마트: DW (Data Warehouse, 중앙 집중식 데이터 집합체; 활용 전 데이터)에서 데이터를 꺼내 사용자에게 제공하는 역할. 
	사용 목적에 맞는 접근성과 사용성을 강조. 대개 특정한 조직 또는 팀에서 사용하며 전략적이고 명확한 목표를 가짐
데이터모델링: 현실 세계의 데이터를 컴퓨터 세계의 데이터베이스에 옮기는 변환 과정 / 추상화, 모형화, 단순화, 명확화
개념적 데이터모델링: 조직, 사용자의 요구사항을 찾고 분석하며, 특정 자료의 중요도와 유지 여부 결정 / ERD
논리적 데이터모델링: 업무의 구체적인 흐름. 비즈니스 정보의 논리적 구조와 규칙을 명확히 표현하는 기법
물리적 데이터모델링: 논리 데이터 모델이 데이터저장소로서 어떻게 컴퓨터 하드웨어에 표현될 것인가를 다룸
노드 인증: 관리 서버와 데이터 콜렉터가 보안 방식으로 서로 통신하는지 확인하는 기술
Repllication(복제): 재해 시 최신 데이터를 복원할 수 있게, 다른 위치로 데이터를 복사하는 프로세스
API (Application Programming Interface): 소프트웨어 시스템간 연결하는 방식
UAT (User Acceptance Testing): 고객이 직접 시스템을 테스트하여 요구사항을 충족하는지 확인하는 단계
회귀테스트 (Regression Test): 새로운 기능 추가나 버그 수정후, 기존 기능들이 여전히 제대로 작동하는지 확인하는 테스트
Go-Live: 공식적으로 런칭되어 실제 사용되기 시작하는 시점
SLA (Service Level Agreement): 업체가 시스템을 어떻게 지원할 것인지 정의한 계약
Patch: 문제를 해결하거나 소소한 기능을 추가하기 위한 비교적 작은 업데이트 / 보안 강화, 버그 수정, 성능 개선 등을 위해 주기적으로 적용
Scalability (확장성): SI 시스템이 사용자나 데이터를 처리할 수 있는 능력
무결성: 데이터 값이 정확한 상태
정합성: 데이터의 정확 유무와 별개로 데이터들의 값이 서로 일치하는 상태
클라우드 컴퓨팅: 인터넷을 통해 컴퓨팅 자원을 필요에 따라 사용할 수 있게 해주는 서비스 / 서버, 저장소, DB 등 포함
NAC (Network Access Control): 엔드포인트 보안기술 / 사용자 또는 시스템 인증 및 네트워크 보안을 통합하는 보안 접근 방식
NAT (Network Address Translation): 사설 IP를 공인 IP로 변환하는 서비스
Co-location Service: 기업이나 사용자가 서버를 구매해서 서버업체에 위탁,관리를 의뢰하여 서버, 회선비만 지불하고 서버를 관리하는 서비스
RDMA (Remote Direct Memory Access): 물리적으로 다른 여러대 서버의 메모리를 가상화시켜 하나의 거대한 메모리 풀로 만들어주는 기술
	메모리와 서버가 직접 통신하기 때문에 DB, 가상머신 마이그레이션, DW, 분석 등 성능을 대폭 향상
MDD ( Model-Driven Development): SW 개발 시 모델을 작성하면 프로그램 코드뿐 아니라 다양한 문서를 자동으로 생성하는 개발 방식
OSI 7계층: 네트워크에서 통신이 일어나는 과정을 7단계로 나눈 것
1. 물리 계층(Physical Layer): 하드웨어 시스템으로 구현(2계층부터는 SW적으로 구현) / 전기적, 기계적, 기능적 특성을 이용해 통신 게이블로 데이터 전달
	사용되는 통신 단위는 Bit(1또는0) / 데이터 전달만 하며 데이터가 무엇인지, 어떤 상태인지 신경쓰지 않음 / 통신 케이블, 리피터, 허브
2. 데이터링크 계층(DataLink Layer): 물리계층을 통해 송수신 되는 정보의 오류와 흐름을 관리하여 정보 전달을 수행. 오류 검출, 재전송, 흐름제어
	맥 주소를 가지고 통신. 전송되는 데이터 단위를 프레임 / 브리지, 스위치
3. 네트워크 계층(Network Layer): 주소(IP)를 정하고 설정한 경로(Route)에 따라 패킷을 전달 / 데이터를 목적지까지 가장 안전하고 빠르게 전달(라우팅)
	전달 경로 선택은 미리 정해지는 정적(Static), 현재 부하 상태에 따라 결정되는 동적(Dynamic) 방식으로 구분 /전송 데이터를 패킷(Packet)
4. 전송 계층(Transport Layer): 통신 활성화 계층 / TCP프로토콜을 이용, 포트를 열어 응용프로그램들이 전송할 수 있게 함 
	TCP: 신뢰성, 연결 지향적 / UDP: 비신뢰성, 비연결성, 실시간 / 받은 데이터를 합쳐 5계층에 보냄
	송신, 수신 프로세스를 직접 연결하는 단대단(End-to-End) 통신 기능. 패킷의 전송이 유효한지 확인하고, 전송 실패된 패킷을 다시 전송
5. 세션 계층(Session Layer): 데이터가 통신하기 위한 논리적인 연결(대문 역할) / 세션 설정, 유지, 종료, 전송 중단 시 복구 기능
	전송 계층과 유사하나, 사용자에게 원격 파일 전송이나 원격 로그인 같은 상위적 연결 개념인 세션 기능을 제공
	송수신 호스트 사이의 대하 제어를 비롯해 상호 배타적인 동작을 제어하기 위한 토근 제어, 일시적 전송 장애를 해결하기 위한 동기(Synchronization) 기능 제공
6. 표현 계층(Presentation Layer): 데이터의 의미(Semantic), 표현방법(Syntax)를 처리. 데이터의 표현 방법이 다르면 서로 데이터를 이해할 수 있게 변환
	데이터 암호화, 인코딩, 명령어 포장, 압축 등 / JPEG, MPEG, GIF, ASCII 등
7. 응용 계층(Application Layer): 최종 목적지. 응용 프로세스와 직접 관계하여 일반적인 응용 서비스 수행(explore, chrome 등)
	허가된 사용자가 적절한 접근 권한에 따라 사용할 수 있어야 함 / HTTP, FTP(File Transfer Protocol), SMTP, Telnet 
그리드 컴퓨팅: WAN (Wide Area Network, 원거리 통신망)으로 연결된 서로 다른 기종의 컴퓨터들을 묶어 가상의 대용량 고성능 컴퓨터를 구성하여
	고도의 연산 작업 혹은 대용량 처리를 수행하는 것
TCP/IP 4계층
1. 네트워크 액세스 계층(Network Access Layer) = 물리(1)+데이터링크 계층(2) / TCP/IP패킷을 네트워크 매체와 송수하는 과정 담당
에러 검출, 패킷의 프레임화 / 네트워크 접근방법, 프레임 포맷, 매체에 대해 독립적 동작하게 설계 /물리적 주소로 MAC 사용 / LAN, 패킷망 등에 사용됨
2. 인터넷 계층(Internet Layer) = 네트워크 계층(3) / 주소지정, 패키징, 라우팅 기능 / 네트워크상 최종 목적지까지 정확히 연결하는 기능 / IP, ARP, RARP
3. 전송 계층(Transport Layer) = 전송 계층(4) / IP와 Port를 이용해 프로세스와 통신 / 응용계층의 세션과 데이터그램 통신 서비스 제공
동신 노드간 연결 제어, 신뢰성 있는 데이터 전송 담당 / TCP, UDP
4. 응용 계층(Application Layer) = 세션(5)+표현(6)+응용계층(7) / 프로그램(브라우저)가 직접 수행하는 레이어, 데이터를 처음으로 받는 곳
다른 계층의 서비스에 접근할 수 있게 하는 앱 제공 / 앱들이 데이터를 교환하기 위해 사용하는 프로토콜 정의
TCP/UDP 기반 응용 프로그램 구현시 사용 / FTP, HTTP, SSH
